底層原理

兩個獨立問題：怎麼接、用多大權限接

🔌
通道 = 水管
內建連接器 ・ MCP ・ CLI ・ 直接操控畫面
——決定「怎麼接」
🔑
授權 = 水管裡的水
免鑰匙 ・ API Key／Token ・ OAuth
——決定「給多大權」
你會問「這條 CLI 該配 Token 還是 OAuth」，但不會問「該用 CLI 還是 Token」——那是兩件不同的事。
階梯 A ・ 鑰匙（授權）

由「免鑰匙」到「OAuth」：隱性 → 顯性

免鑰匙
靠本機權限／你已登入的狀態｜白話：它站在你已經開好的門裡（Obsidian、操控畫面）
API Key／Token
一把萬能長期鑰匙，貼了就全開、要手動撤｜白話：家裡大門鑰匙（資料庫、Notion）
OAuth
限範圍、會過期、隨時撤｜白話：門禁卡（Google Workspace、gh、rclone）

安全上：能用 OAuth 就別到處貼 API Key。（註：免鑰匙不是陽春，是一種正當方式）

階梯 B ・ 通道（怎麼接）

由「內建連接器」到「操控畫面」：隱性 → 顯性

內建連接器
原廠接好，開關即用
MCP
萬用轉接頭，社群做一堆
CLI
跑命令列驅動（gh／rclone），還能自造（CLI-Anything）
直接操控畫面
沒 API 也能連，最後手段

CLI 是第四種通道——就是 EP03 的「跑程式」指向外部工具；打包成 Skill 的玩法留到 EP05。

2026 現況：內建連接器底層幾乎全是 MCP（Claude Connectors ・ ChatGPT Apps ・ AntiGravity MCP Store）——差別只在誰幫你接；OpenCode 則純手寫設定檔，最自由也最自己來。

搭配圖 ・ 兩條階梯怎麼接起來

通道配鑰匙：MCP 最靈活，首尾卻對調

粉色線＝首尾對調：最隱性的連接器配最顯性的 OAuth、最顯性的操控畫面配最隱性的免鑰匙。會交叉，正好證明這是兩條獨立的軸。

第一大段 ・ 非 Google

非 Google：簡單路貼鑰匙，完整路走 OAuth

🟢 簡單路
內建連接器：原廠接好（如 Codex 的 GitHub 連接器）。
MCP ＋ API Key：有人做了 server，貼把鑰匙就接。最原始、最快。
🔒 完整控制路
MCP ＋ OAuth：比 API Key 安全，登入一次、範圍受限、可撤銷。
本地直存：像 Obsidian 讀本機筆記，免 token，靠檔案權限。
有現成連接器就用，沒有就 MCP 貼把鑰匙；要更安全，把鑰匙換成 OAuth。

第二大段 ・ Google 家

Google 的東西，要走 Google 的門

🎓
老師世界一大半在這
Classroom 發作業、Drive 存教材、Gmail 寄通知、行事曆排課。
🚫
破除迷思
以為「一組 API Key 就能讀我的 Classroom」？不行。個人資料一律要 OAuth 或 Service Account。
Google Workspace 個資服務不給你簡單 API Key 就存取個人資料——這是跟非 Google 最大的不同。
Google 家服務總覽 ・ 本段核心

連 Google 自己家，接法都不一樣

服務老師拿來做什麼接法鑰匙
Gmail自動寄通知／回信連接器 ／ 自建 OAuthOAuth
行事曆排課表／排程連接器 ／ 自建 OAuthOAuth
雲端硬碟讀教材、整理講義、產出簡報連接器 ／ 自建 OAuthOAuth

上 4 個是 Workspace 個資 → 一律 OAuth；下 2 個是 Google 的開發／AI 工具 → 各走各的。

Google（Workspace）・ 四種方式 + 何時自建 OAuth

Google Workspace 接法比較：不要把 API Key 當萬能鑰匙

方式適合情境說明
原生連接器零設定讀取OAuth 在底層跑，你只按開關；範圍被原廠框住
自建 GCP ＋ OAuth寫入／細緻 scope要發作業、寄信、建檔就走這條
Service Account伺服器自動化適合共用資料或網域委派；碰不到個人 Gmail／Drive
API Key公開資料／不碰個資不能讀 Drive、Gmail、Classroom 個人資料
原生連接器＝零設定但受限；自建 OAuth＝要花工但完全掌控。（設定步驟之後做懶人包）
第三段 ・ 最後手段

直接操控畫面：沒 API 也能連，但別當首選

🖱️
怎麼運作
讓 Agent 像真人點滑鼠、打字（Computer Use／瀏覽器操控／Chrome 外掛）。授權靠你已登入的 session，不用 token。
⚠️
萬用但脆
畫面一改就可能失手、速度慢。是最後手段，不是首選。
能用連接器或 MCP，就別讓它硬點畫面。

收尾 2 ・ 安全

手腳伸出電腦，鑰匙也交了出去

🔑
看清楚哪把鑰匙
API Key 是萬能長期鑰匙、OAuth 範圍受限可撤銷——能 OAuth 就別到處貼 Key。
🧑‍🎓
學生資料去識別化
接 Classroom／資料庫尤其注意，只存座號＋班級代號。
🚫

MCP server／連接器會拿到你的授權，跟裝來路不明外掛一樣要小心。
不用的 token 要撤；API Key 限制用途；OAuth scope 只開需要的。
