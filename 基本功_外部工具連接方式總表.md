# 基本功外部工具連接方式總表（AntiGravity／Codex 路線）

> 範圍：以 AntiGravity 與 Codex 基本功實際出現過的外部工具連接路線為主，相關的 Claude／GoogleAI 基本功素材一併標註。
> 本表不整理 Google 雲端／GCP 細部設定，也不整理 Gemini Canvas。

| 路線 | 適合情境 | 主要通道 | 授權方式 | 代表影片 |
|------|----------|----------|----------|----------|

## AntiGravity 路線

### 1. GitHub + Firebase：正式互動工具路線

| 項目 | 說明 |
|------|------|
| 用途 | 網頁互動工具、課堂遊戲、需要資料庫記錄的作品 |
| 通道 | GitHub repo 管版本與部署素材；Firebase 當資料庫或後端服務 |
| 授權 | GitHub 需要帳號登入；Firebase 需要專案設定與服務設定 |
| 優點 | 比較完整、可長期維護，適合正式作品 |
| 代價 | GitHub 註冊與 Firebase 設定對新手較難 |
| 代表素材 | AntiGravity基本功 EP05；Codex基本功 EP04；Claude基本功 EP09.5 |

口訣：**要做正式互動工具，GitHub 管版本，Firebase 管資料。**

### 2. Netlify + GAS：簡化部署替代路線

| 項目 | 說明 |
|------|------|
| 用途 | 前端網頁部署、簡易資料儲存、用試算表或 Apps Script 做後端 |
| 通道 | 前端部署到 Netlify；後端用 GAS / Apps Script Web App |
| 授權 | Netlify Personal Access Token；Google 帳號授權 clasp / Apps Script |
| 優點 | 對新手通常比 GitHub + Firebase 更容易起步 |
| 代價 | 後端能力與維護彈性不如完整資料庫；Google 帳號授權步驟仍要處理 |
| 代表素材 | AntiGravity基本功 EP03 |

口訣：**前端丟 Netlify，簡易後端交給 GAS。**

### 3. Google Classroom：教學平台代理路線

| 項目 | 說明 |
|------|------|
| 用途 | 發作業、讀取課程與學生名單、整理繳交與成績相關資料 |
| 通道 | Classroom API / Classroom Agent Kit |
| 授權 | OAuth，通常需要登入要操作的教育帳號 |
| 優點 | 直接打到老師日常教學平台，教學價值高 |
| 代價 | OAuth 與學校網域權限可能卡關，寫入操作需要更謹慎 |
| 代表素材 | AntiGravity基本功 EP06 |

口訣：**讀 Classroom 要授權，寫 Classroom 更要確認權限範圍。**

### 4. Padlet：課程牆快速生成路線

| 項目 | 說明 |
|------|------|
| 用途 | 一句話生成分區課程牆、投票牆、AI 圖文素材 |
| 通道 | Padlet MCP / API |
| 授權 | Padlet token |
| 優點 | 展示效果明顯，很適合課堂活動 |
| 代價 | token 要保管；Padlet 介面或權限變動時需重新確認 |
| 代表素材 | AntiGravity基本功 EP07 |
| 可引用模板 | `G:\我的雲端硬碟\2026邀約\Padlet模組` |

口訣：**Padlet 用 token 接，適合快速做課堂牆。**

## Codex 路線

### 1. GitHub + Obsidian：程式倉庫與第二大腦路線

| 項目 | 說明 |
|------|------|
| 用途 | 讀寫 GitHub repo、同步工作流、把踩坑與進度寫回 Obsidian |
| 通道 | Codex GitHub connector / GitHub plugin、gh CLI、Obsidian MCPVault |
| 授權 | GitHub 登入 / gh auth；Obsidian 走本機檔案權限，通常免 token |
| 優點 | 最符合長期教學專案維護，能把 repo 與第二大腦串起來 |
| 代價 | GitHub connector 與 gh CLI 要分開確認；Obsidian vault 路徑要正確 |
| 代表素材 | Codex基本功 EP02；Codex基本功 EP03 |

口訣：**GitHub 管程式，Obsidian 管工作流。**

### 2. Firebase：Codex 互動工具後端路線

| 項目 | 說明 |
|------|------|
| 用途 | 課堂遊戲、互動工具、需要記分或資料同步的前後端專案 |
| 通道 | Firebase 設定檔、MCP / CLI、專案初始化 skill |
| 授權 | Firebase 專案服務設定 / 金鑰 |
| 優點 | 適合做需要後端的教學應用 |
| 代價 | 金鑰與規則要小心，不能把敏感設定直接公開到前端 |
| 代表素材 | Codex基本功 EP04；Claude基本功 EP09.5 |

口訣：**Codex 做互動工具，Firebase 負責存資料。**

### 3. 外掛 + Skills：能力安裝與流程打包路線

| 項目 | 說明 |
|------|------|
| 用途 | 補齊瀏覽器、文件、簡報、試算表、GitHub 等能力；把常用工作流打包成一句話 |
| 通道 | Codex plugins、skills、本機 skill 資料夾 |
| 授權 | 多數走 Codex 帳號與本機檔案；若外掛連外部服務，依該服務授權 |
| 優點 | 能把教學素材製作流程固定下來，降低每次重講需求 |
| 代價 | 外掛是否可寫入、skill 是否正確觸發，要逐項驗證 |
| 代表素材 | Codex基本功 EP02；Codex基本功 EP04 |

口訣：**外掛補能力，Skill 打包流程。**

## 四家 Agent 的連接器入口對照（2026-07 查證）

> 關鍵觀念：2026 年的「內建連接器」底層幾乎全是 MCP——連接器＝**原廠幫你裝好、審核過、OAuth 接好的 remote MCP**。四階通道不是互斥技術，是「你要自己動手多少」的光譜。

| Agent | 連接器入口 | 實際形式 | 手動 MCP 設定 |
|-------|-----------|---------|--------------|
| Claude Code／Desktop | Settings → Connectors（目錄 500+） | 審核過的 remote MCP ＋ OAuth；可貼 URL 加自訂 | `claude mcp add` ／ `.mcp.json` |
| Codex／ChatGPT | Settings → Apps（原名 connectors） | MCP（Apps SDK）＋ OAuth；ChatGPT 連好 Codex 共用 | `codex mcp add` ／ `~/.codex/config.toml` |
| AntiGravity | IDE 內建 MCP Store（一鍵安裝） | MCP（streamable HTTP），Google 服務為主；另內建瀏覽器 `/browser` | MCP 設定檔（CLI／IDE 共用） |
| OpenCode | 無目錄（最 DIY） | — | `opencode.json` 的 `mcp` 區塊（local／remote）＋ plugins |

口訣：**越往下，越自由、也越自己來。**

## 安全檢查口訣

| 類型 | 要檢查什麼 |
|------|------------|
| Token / API Key | 不用就撤、用途要限制、不要貼到公開 repo |
| OAuth | scope 只開需要的，寫入權限要特別確認 |
| 本地直存 | vault / 專案資料夾路徑要明確，不要讓 Agent 掃到無關資料 |
| 資料庫 / 後端 | 規則要檢查，學生資料要去識別化 |
| 部署 | 分清楚前端公開檔案與後端秘密設定 |

總口訣：**通道決定怎麼接，授權決定給多大權；能撤的要會撤，會公開的不要放秘密。**
